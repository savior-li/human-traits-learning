# Contributing to Human Traits Learning Skill

We welcome contributions to enhance this skill! This document provides guidelines for contributing to the `human-traits-learning` skill.

## Getting Started

1. **Fork the repository** on [GitHub](https://github.com/your-username/human-traits-learning) or [Gitee](https://gitee.com/your-username/human-traits-learning)
2. **Clone your fork**:
   ```bash
   git clone https://github.com/your-username/human-traits-learning.git
   cd human-traits-learning
   ```
3. **Create a feature branch**:
   ```bash
   git checkout -b feature/your-feature
   ```

## Development Guidelines

### Core Principles
- **Three Red Lines Compliance**: All code must adhere to:
  1. **Closure Awareness**: Provide verification evidence for all functionality
  2. **Fact-Driven**: Validate assumptions with actual testing/tooling
  3. **Exhaustive Problem-Solving**: Consider multiple approaches before implementation

### Code Structure
```
human-traits-learning/
├── SKILL.md              # Skill specification and usage guide
├── README.md             # Multi-platform documentation
├── LICENSE               # MIT License
├── package.json          # npm package configuration
├── .gitignore            # Version control ignore rules
├── CONTRIBUTING.md       # This file
├── scripts/              # Core JavaScript implementations
│   ├── apply-historical-lessons.js    # Historical learning engine
│   ├── corporate-training.js          # Corporate training framework  
│   └── leadership-assessment.js       # Leadership readiness assessment
├── data/                 # Training data and patterns
│   └── human_behavior_patterns.json   # Human behavior pattern library
└── examples/             # Usage examples and demonstrations
    └── usage_examples.md              # Practical implementation examples
```

### Testing Requirements
All contributions must include:
- **Unit tests** for core functionality
- **Integration tests** demonstrating real-world usage
- **Verification evidence** showing successful execution
- **Performance benchmarks** for complex operations

### Documentation Standards
- Update `SKILL.md` with any new API changes
- Update `README.md` with new features or usage patterns
- Add examples to `examples/` directory for new functionality
- Document any data format changes in `data/` directory

## Contribution Areas

### High Priority
- **Enhanced Historical Learning Algorithms**: More sophisticated pattern recognition
- **Additional Corporate Training Models**: Different industry/company approaches
- **Domain-Specific Best Practices**: Industry-specific excellence standards
- **Performance Optimization**: Faster execution and lower resource usage

### Medium Priority  
- **Extended Assessment Frameworks**: More granular evaluation metrics
- **Multi-Language Support**: Internationalization for global usage
- **Visualization Tools**: Dashboards and progress tracking
- **Integration Adapters**: Compatibility with other AI frameworks

### Low Priority
- **Documentation Improvements**: Better examples and explanations
- **Code Quality**: Refactoring and optimization
- **Testing Coverage**: Additional test cases and scenarios

## Pull Request Process

1. **Ensure compliance** with three red lines principles
2. **Run all tests** and verify they pass
3. **Update documentation** for any changes made
4. **Provide verification evidence** of working functionality
5. **Submit pull request** with clear description of changes
6. **Respond to feedback** promptly and thoroughly

## Code of Conduct

This project follows the principles of high-performance culture:
- **Professional Excellence**: Maintain high standards in all contributions
- **Constructive Feedback**: Provide helpful, specific feedback to others
- **Continuous Improvement**: Always look for ways to improve quality
- **Collaborative Spirit**: Work together to achieve shared goals
